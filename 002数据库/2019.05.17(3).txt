1.数据库的完整性
  1.数据正确
    例如：性别只能是男和女
  2.数据相容
    例如：学生所选的课程必须是存在的课程
2.完整性和安全性
  完整性
    防范对象：不合语义的、不正确的数据
  安全性
    防范对象：非法用户和非法操作
  完整性是阻止合法用户通过合法操作向数据库中加入不正确的数据
  安全性防范的是非法用户和非法操作存取数据库中的正确数据
3.完整性
  关系模型的实体完整性
            参照完整性
            用户定义完整性
4.数据完整性检查机制
  检查时机：insert,update,delete，数据提交时 检查
  
  违约处理：1.拒绝
            2.级联  去除掉违约的条件
5.DBMS进行完整性检查
6.实体完整性
  1.create table 中用 primary key定义
    列级约束条件  一个主码
    表记约束条件  一个或多个主码

    主码唯一且不为空
    因为主码是唯一的，所以添加主码是要看它是否唯一
       如果没有索引，对于大表来说，遍历是非常费时的
       所以自动给主码加上索引（例如B+述索引，数据结构中能学到）
       减少判断时间
7.RDBMS完整性控制机制的3个组成部分
  定义机制
  检查机制  自动
  违约处理机制
8.参照完整性
  外码或者为空
      或者为其它关系（基本表）的主码
  foreign key peferences
  可以是列级约束，也可以是表记约束
9.参照完整性的检查（自动）
  表的增、删、改
10.参照完整性约束的违约处理（自己设定）
   拒绝、级联、设置为空值
   foreign key (Sno) references Course(Sno)
       on delete cascade
       on update cascade

   其它
       on delete no action
       on delete set null
11.用户定义的完整性
   属性上的约束条件
   元组上的约束条件
12.属性上的约束条件（写在列名 列类型 之后）
   not null  不允许取空值
   unique    列值唯一
   check     指定列值应该满足的条件
             check(Grade>=0 and Grade<=100)

   可以写  unique not null
13.用户完整性约束的检查条件（属性上）
   插入，修改
14.用户完整性约束的违约处理（属性上）
   拒绝
15.元组上的约束条件（多属性联合约束）
   表级约束
   check(Ssex='女' or Sname not like 'Ms.%')

   触发条件：插入，修改
   违约处理：拒绝
16.完整性小结
   实体完整性和用户定义完整性
      定义方法：create table
      检查时机：插入、修改
      违约处理：拒绝
   参照完整性
      定义方法：create table
      检查时机：参照表：插入、修改
                被参照表：删除、修改
                （都是对主码和外码）
      违约处理：拒绝、级联、设为空值
